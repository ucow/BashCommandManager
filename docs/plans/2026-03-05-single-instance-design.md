# 单例启动模式设计方案

## 概述
实现应用的单例启动模式，当双击exe启动时，如果已有正在运行的应用，则激活已运行的应用窗口而不是启动新实例。

## 需求
- 当检测到已有实例运行时，激活已有实例的窗口
- 如果窗口最小化到托盘，则恢复并显示窗口
- 新启动的实例在通知主实例后退出

## 架构设计

```
新实例启动
    ↓
Mutex检测（全局命名互斥锁）
    ↓
是否已有实例？
    ├─ 否 → 正常启动，创建命名管道服务器监听
    └─ 是 → 连接命名管道发送"激活"消息 → 新实例退出

主实例（监听中）
    ↓
收到"激活"消息
    ↓
在主线程激活窗口（Show + Activate）
```

## 组件设计

### 新增组件
- `Services/SingleInstanceService.cs` - 单例检测和IPC通信服务

### 修改组件
- `App.xaml.cs` - 启动时调用单例检测逻辑
- `MainWindow.xaml.cs` - 添加处理激活消息的方法

## 技术方案

使用 **Mutex + 命名管道（NamedPipe）**：
- `Mutex`: 检测是否已有实例在运行
- `NamedPipeServerStream`: 主实例监听激活请求
- `NamedPipeClientStream`: 新实例发送激活命令

## 错误处理

- 命名管道连接超时：静默退出
- 互斥锁创建失败：记录日志，允许启动（降级）
- 消息处理异常：捕获异常，不影响主程序

## 测试方案

1. 双击exe启动第一个实例 → 应用正常启动
2. 再次双击exe → 第一个实例的窗口被激活
3. 窗口最小化到托盘后双击exe → 窗口从托盘恢复并激活
